# プロジェクト憲章

プロジェクト憲章を書きます。

## プロジェクトを立ち上げるきっかけ

システム開発では、要件定義や設計書の品質がプロジェクト全体の品質を左右する。しかし、それらを適切に作成するには経験や知識が必要であり、多くの開発者にとって大きな障壁となっている。

また、生成AIを活用した開発では、会話を重ねるにつれて当初の目的や前提条件が薄れ、本来実現したかったシステムとのズレが生じることがある。

このズレを防ぐためには、要件や設計内容を継続的に整理し、知識として蓄積することが重要である。しかし、利用者自身がMarkdownや設計書を作成・更新し続けることは大きな負担となる。

そこで本プロジェクトでは、AIとの対話を通じて思考を整理し、知識を構造化し、要件定義から設計書までを継続的に育てることができるプラットフォームを構築する。

## プロジェクト理念

AIは利用者の代わりに考えるものではない。

AIは利用者がより良く考えられるよう導く存在である。

本プロジェクトでは、AIを単なる質問応答ツールとしてではなく、システムアナリストとして位置付ける。

AIは利用者との対話を通じて思考を整理し、不足している考慮事項を提示し、知識を構造化することで、誰もが一定品質の要件定義・設計を行える環境を提供する。

本プロジェクトの目的は「設計書を作ること」ではない。

利用者が「考えるべきこと」を考えられるようになることを支援することである。

AIは設計書を作るだけではなく、プロジェクトの現在地を示し、次に考えるべきことを導く。

AIはプロジェクトを共に育てるパートナーとして、利用者の思考を支え続ける存在を目指す。
