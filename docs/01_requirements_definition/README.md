# Phase2（Requirements Definition）

## Phase2の目的

基本設計を開始できるレベルまで、
システムの目的・業務・機能・制約・非機能要件を整理する。

## Phase2の完了条件

- Project Charter が完成している
- 要件定義書（Markdown）が完成している
- 要件定義書をGitHubで管理している
- 基本設計へ進むための前提条件が整理されている
- 未確定事項・課題・将来検討事項が明確になっている
- AIシステムアナリストが利用者へ必要十分な問いかけを行い、Markdownとして整理できる構成になっている

## 成果物

- docs/00_project_charter
- docs/01_requirements_definition
- Phase2成果報告書（PDF）

## 備考

要件定義は固定化するものではない。

基本設計・詳細設計・実装を通じて必要に応じて更新し、プロジェクトとともに継続的に育てていく。
